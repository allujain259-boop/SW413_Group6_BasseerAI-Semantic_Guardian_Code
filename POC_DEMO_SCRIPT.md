# 🎬 سيناريو العرض للجنة التحكيم
## Judges Demo Script — Basseer AI (3–5 Minutes)

---

## ⏱️ الدقيقة 0:00 – المقدمة (30 ثانية)

**قل:**
> "بصير AI هو نظام ذكي للكشف التلقائي عن التناقضات المنطقية والدلالية في بيانات الاستبيانات الحكومية.
> يجمع بين محرك قواعد فوري وذكاء اصطناعي تحليلي لضمان جودة البيانات في الوقت الحقيقي."

**أظهر:** الواجهة الرئيسية مع الشعار وشريط الحالة الذي يوضح وضع LLM.

---

## ⏱️ الدقيقة 0:30 – تجربة الإدخال مع تناقض واضح (60 ثانية)

**Tab: 📝 إدخال الاستبيان**

1. أدخل البيانات التالية:
   - العمر: **19**
   - سنوات الخبرة: **15**
   - التعليم: **PhD**
   - الوظيفة: **Doctor**
   - الدخل: **80,000**
   - المنطقة: **الرياض**

2. اضغط **"تحقق من البيانات"**

**أبرز النقاط:**
- درجة الثقة المنخفضة (تظهر بالأحمر)
- التحذيرات مع أيقونات الشدة (🔴 حرجة)
- مصدر الكشف: "📏 القواعد" أو "🤖 الذكاء" أو "🔀 هجين"
- التوصيات التلقائية مع اقتراحات تصحيح
- السبب بالعربية والإنجليزية (Explainable AI)
- زمن الاستجابة بالميلي ثانية

3. الآن أدخل بيانات نظيفة (عمر 35، خبرة 10، بكالوريوس) → أظهر النتيجة الخضراء 100%

---

## ⏱️ الدقيقة 1:30 – الباحث الميداني (45 ثانية)

**Tab: 🔍 الباحث الميداني**

**قل:**
> "هذه الواجهة مخصصة للباحث الميداني لمراجعة الاستجابات المشبوهة."

**أظهر:**
- قائمة السجلات مرتبة حسب درجة الثقة (الأقل أولاً)
- تصفية حسب الحالة (بانتظار المراجعة / مُصعّد / تم التأكيد)
- فتح سجل وعرض التفاصيل والتحذيرات
- أزرار الإجراءات الأربعة: مراجعة ← تأكيد / طلب تصحيح / تصعيد
- اضغط على **"✅ تأكيد"** أو **"🔺 تصعيد"** لإظهار تغيّر الحالة

---

## ⏱️ الدقيقة 2:15 – لوحة الجهة المشرفة (90 ثانية)

**Tab: 📊 لوحة الجهة المشرفة**

### مؤشرات الأداء (15 ثانية)
- إجمالي السجلات، نسبة الجودة، متوسط الثقة، التناقضات، الزمن، بانتظار المراجعة

### الرسوم البيانية (15 ثانية)
- توزيع درجات الثقة (عالية/متوسطة/منخفضة)
- مصدر الكشف (قواعد / ذكاء / هجين) → يوضح القيمة المضافة للذكاء

### الخريطة الحرارية (30 ثانية)

**قل:**
> "الخريطة الحرارية هي الأداة الاستراتيجية الأهم — تكشف أين تتركز التناقضات."

- بدّل بين المقاييس الأربعة: العدد → النسبة → الثقة → **معدل اختلاف الذكاء**
- بدّل المحور: المنطقة ↔ الباحث
- اضغط على خلية ساخنة لعرض التفاصيل:
  - القواعد الأكثر انتهاكاً
  - تفسير الذكاء الاصطناعي
  - التوصية

### التوصيات المجمّعة (15 ثانية)

**أظهر:**
- توصيات إعادة صياغة الأسئلة ذات معدل الخطأ العالي
- توصيات تدريب الباحثين في مناطق محددة
- توصيات مراجعة أداء باحثين معينين

### أداء الباحثين + صحة النظام (15 ثانية)
- جدول ترتيب الباحثين حسب عدد الأخطاء
- مؤشرات صحة النظام: حالة الخادم، الذكاء، الاستدعاءات، الكاش

---

## ⏱️ الدقيقة 3:45 – إعدادات LLM (30 ثانية)

**الشريط الجانبي:**

**قل:**
> "النظام يدعم إعداد مزود الذكاء من داخل التطبيق بدون تعديل كود."

- أظهر صندوق الإعدادات في الشريط الجانبي
- أظهر المزودين الثلاثة: OpenAI / Anthropic / Gemini
- أظهر مؤشرات الصحة: استدعاءات، زمن، كاش

**قل:**
> "إذا فشل الاتصال بالذكاء الاصطناعي، النظام يتحول تلقائياً لوضع القواعد بدون أي توقف."

---

## ⏱️ الدقيقة 4:15 – الخلاصة (30 ثانية)

**قل:**
> "بصير AI يقدم حلاً متكاملاً لضمان جودة بيانات المسوحات:
> - كشف فوري بالقواعد + تحليل عميق بالذكاء
> - توصيات تلقائية لتحسين الاستبيانات
> - ثلاث واجهات لثلاثة أدوار مختلفة
> - خريطة حرارية استراتيجية للقرارات
> - قابل للتشغيل محلياً بدون أي تعقيد"

---

## 📝 ملاحظات تقنية للعرض

- تأكد من تشغيل الخادم قبل العرض: `uvicorn backend.main:app --port 8000`
- تأكد من إدخال البيانات: `python scripts/ingest_dataset.py`
- لوحة المراقبة: `streamlit run dashboard/app.py`
- إذا أردت عرض LLM حقيقي، أعد مفتاح API مسبقاً في إعدادات التطبيق
- Endpoint صحة النظام: http://127.0.0.1:8000/health
- توثيق API التفاعلي: http://127.0.0.1:8000/docs
